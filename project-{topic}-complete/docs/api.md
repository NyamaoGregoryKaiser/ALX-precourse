# API Documentation

The Data Visualization Tools System provides a comprehensive RESTful API for managing all aspects of the application.

## OpenAPI (Swagger UI & ReDoc)

The API is fully documented using OpenAPI (formerly Swagger), and interactive documentation is automatically generated by FastAPI.

You can access the API documentation at:

*   **Swagger UI (Interactive):** `http://localhost:8000/api/v1/docs`
*   **ReDoc (Reader-friendly):** `http://localhost:8000/api/v1/redoc`

### Authentication

To interact with most API endpoints, you need to authenticate using a JWT access token.

1.  **Obtain Access Token:**
    *   Go to `http://localhost:8000/api/v1/docs`.
    *   Expand the `auth` section.
    *   Click on the `POST /api/v1/auth/access-token` endpoint.
    *   Click "Try it out".
    *   Enter your `username` (email, e.g., `admin@example.com`) and `password` (e.g., `adminpassword`).
    *   Click "Execute".
    *   Copy the `access_token` from the response.

2.  **Authorize API Calls:**
    *   Click the green "Authorize" button at the top right of the Swagger UI page.
    *   In the dialog, enter your `access_token` in the format `Bearer YOUR_ACCESS_TOKEN`.
    *   Click "Authorize" and then "Close".

    Now, all subsequent requests made through Swagger UI will include your authentication token.

## API Endpoints Overview

The API is structured under `/api/v1/` and grouped by resource tags:

### 1. `auth`

*   `POST /api/v1/auth/access-token`: Obtain a JWT access token using email and password.
*   `POST /api/v1/auth/test-token-corrected`: Test if a provided JWT token is valid and retrieve the authenticated user's details.

### 2. `users` (Admin/Self-Service)

*   `POST /api/v1/users/`: Create a new user (Superuser only).
*   `GET /api/v1/users/`: Retrieve a list of all users (Superuser only).
*   `GET /api/v1/users/me`: Retrieve the profile of the current authenticated user.
*   `PUT /api/v1/users/me`: Update the profile of the current authenticated user.
*   `GET /api/v1/users/{user_id}`: Retrieve a specific user by ID (Superuser only).
*   `PUT /api/v1/users/{user_id}`: Update a specific user by ID (Superuser only).
*   `DELETE /api/v1/users/{user_id}`: Delete a specific user by ID (Superuser only).

### 3. `datasources`

*   `POST /api/v1/datasources/`: Create a new data source connection.
*   `GET /api/v1/datasources/`: Retrieve a list of data sources (all for superuser, own for normal users).
*   `GET /api/v1/datasources/{data_source_id}`: Retrieve a specific data source by ID.
*   `PUT /api/v1/datasources/{data_source_id}`: Update a specific data source by ID.
*   `DELETE /api/v1/datasources/{data_source_id}`: Delete a specific data source by ID.
*   `POST /api/v1/datasources/{data_source_id}/test`: Test the connection to a data source.

### 4. `datasets`

*   `POST /api/v1/datasets/`: Create a new dataset (query/extraction from a data source).
*   `GET /api/v1/datasets/`: Retrieve a list of datasets.
*   `GET /api/v1/datasets/{dataset_id}`: Retrieve a specific dataset by ID.
*   `PUT /api/v1/datasets/{dataset_id}`: Update a specific dataset by ID.
*   `DELETE /api/v1/datasets/{dataset_id}`: Delete a specific dataset by ID.
*   `POST /api/v1/datasets/{dataset_id}/data`: Fetch and preview the raw data for a dataset. (Cached)

### 5. `visualizations`

*   `POST /api/v1/visualizations/`: Create a new visualization (chart definition based on a dataset).
*   `GET /api/v1/visualizations/`: Retrieve a list of visualizations.
*   `GET /api/v1/visualizations/{visualization_id}`: Retrieve a specific visualization by ID.
*   `PUT /api/v1/visualizations/{visualization_id}`: Update a specific visualization by ID.
*   `DELETE /api/v1/visualizations/{visualization_id}`: Delete a specific visualization by ID.
*   `POST /api/v1/visualizations/{visualization_id}/data`: Fetch and transform data for a visualization, ready for rendering. (Cached)

### 6. `dashboards`

*   `POST /api/v1/dashboards/`: Create a new dashboard (layout of visualizations).
*   `GET /api/v1/dashboards/`: Retrieve a list of dashboards.
*   `GET /api/v1/dashboards/{dashboard_id}`: Retrieve a specific dashboard by ID.
*   `PUT /api/v1/dashboards/{dashboard_id}`: Update a specific dashboard by ID.
*   `DELETE /api/v1/dashboards/{dashboard_id}`: Delete a specific dashboard by ID.
*   `POST /api/v1/dashboards/{dashboard_id}/data`: Fetch all data for visualizations contained within a dashboard. (Cached)
